1.2 최종 서비스 정의
퇴사자가 남긴 날것의 파일들(이메일 백업, 슬랙 내보내기, 메모장, 문서 등)을 업로드하면, AI가 자동으로 내용을 분석하여 프로젝트별 / 거래처별 / 타임라인별로 분류하고 핵심 요약이 담긴 인수인계 대시보드 웹페이지를 자동 생성하는 능동적 시스템입니다.

&apos;질문하지 않아도&apos; AI가 먼저 전체 맥락을 파악하고 구조화
신규 인수자는 대시보드만 열면 퇴사자의 업무 전체를 한눈에 파악 가능
챗봇, Vector DB, RAG 파이프라인 — 전부 불필요

3.2 Supabase DB 스키마
벡터 없이 일반 PostgreSQL 테이블 2개만 사용합니다. Supabase SQL Editor에서 최초 1회 실행합니다.

-- 1. 원본 문서 테이블 (업로드된 파일 원문 저장)CREATE TABLE documents (  id          BIGSERIAL PRIMARY KEY,  file_name   TEXT,           -- 원본 파일명  source_type TEXT,           -- &apos;email&apos; | &apos;slack&apos; | &apos;docx&apos; | &apos;pdf&apos; | &apos;txt&apos;  raw_content TEXT,           -- 추출된 원문 텍스트  created_at  TIMESTAMPTZ DEFAULT NOW());-- 2. AI 분류 결과 카드 테이블CREATE TABLE cards (  id          BIGSERIAL PRIMARY KEY,  project     TEXT,           -- AI가 추출한 프로젝트명  client      TEXT,           -- AI가 추출한 거래처명  event_date  DATE,           -- AI가 파싱한 날짜  summary     TEXT,           -- AI 3줄 요약  source_type TEXT,           -- 출처 포맷  doc_id      BIGINT REFERENCES documents(id),  created_at  TIMESTAMPTZ DEFAULT NOW());

3.3 GPT-4o 프롬프트 설계
각 문서에 대해 아래 형식의 JSON 응답을 강제합니다. 응답은 파싱하여 cards 테이블에 저장합니다.

SYSTEM: &quot;다음 문서를 분석하여 JSON만 반환하라. 다른 텍스트 없이.&quot;USER: f&apos;&apos;&apos;문서 내용: {raw_text[:3000]}아래 JSON 형식으로 분석 결과를 반환하라:{  &quot;project&quot;:    &quot;프로젝트명 (불명확하면 null)&quot;,  &quot;client&quot;:     &quot;거래처/고객사명 (없으면 null)&quot;,  &quot;event_date&quot;: &quot;YYYY-MM-DD (불명확하면 null)&quot;,  &quot;summary&quot;:    &quot;핵심 내용 3줄 요약 (한국어)&quot;}&apos;&apos;&apos;

7.2 정성적 목표
신규 인수자가 퇴사자 문서를 직접 읽지 않고도 업무 맥락을 파악할 수 있는 수준의 대시보드 완성
파일 포맷에 무관하게 &apos;폴더째 드래그앤드롭&apos;하면 대시보드가 생성되는 사용자 경험 구현
Glean 등 엔터프라이즈 솔루션과 동일한 핵심 가치를 무료 인프라로 팀 단위에서 실현
